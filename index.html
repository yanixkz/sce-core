What SCE Core is now

SCE Core is product-first and reusable: a decision engine with inspectable API/UI/graph surfaces, packaged through flagship demos, grounded in a CDS → SCE theory bridge, and extended by an explicit research program.

Product surface

Use endpoints such as /decide, /memory, /reliability, /graph, and /ui as integration surfaces.

Flagship windows

supplier-risk is practical and operator-facing; hypothesis is research-facing. Both run on the same runtime loop.

Theory + research continuity

CDS concepts map to implemented mechanisms, and open problems stay linked to the same shipping engine.

Core loop

The engine stays centered on one inspectable cycle.

Decide → Explain → Remember / Reliability → Improve

Decide

Generate and rank candidate trajectories under constraints.

Explain

Expose decision backbone versus dangling context for structural clarity.

Remember / Reliability → Improve

Track prediction error, retain episodic traces, and alter the next selection pressure.

What makes it different

SCE differentiates by coupling a decision backbone with measurable adaptation signals and inspectable surfaces.

Decision backbone
forward  = nodes reachable from evidence
backward = nodes that can reach the target decision
backbone = forward ∩ backward
dangling = forward - backbone

Structural extraction shows which facts carried the decision, not just narrative justification.

Reliability-aware planning
base score
+ memory bias
+ remembered reliability
→ selected plan

Reliability is tracked from outcome quality and fed back into reselection, not treated as generic confidence.

Episodic memory + inspectability

Remembered episodes affect later choices, while API/graph/UI views keep state observable for debugging and audits.
